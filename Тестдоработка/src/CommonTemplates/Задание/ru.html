Задание

Для успешного прохождения нужно выполнить первые 2 задачи. Задача №3, на логику, не обязательна, но будет дополнительным плюсом при выборе кандидата.
При проверке задания, будет обращено внимание на чистоту кода и соответствие стандартам разработки 1С. 
  1. В справочнике Клиенты реализовать контроль уникальности номера мобильного телефона. Система не должна давать записать элемент справочника, если введенный номер есть у другого клиента.
    Для определения признака мобильного номера можно воспользоваться функцией ЭтоМобильныйНомерТелефона() общего модуля ОбщегоНазначенияЕАПТЕКАКлиентСервер.
    Предусмотреть следующее - пользователь может ввести номер произвольным образом. Например, номера "9152147455" и "+7 (915) 214-7455" являются одинаковыми.
  2. Реализовать проведение документа Заказ клиента. Документ должен делать движения по регистру накопления Товары на складах - свободный остаток должен быть уменьшен, резерв увеличен. При проведении должен работать контроль отрицательных остатков. Заказ не должен быть проведен, если хотя бы одной позиции не хватает. 
  3. *Дополнительное задание. Во встроенной обработке МаксимальнаяСуммаПодмассива в основной форме реализовать тело функции МаксимальнаяСумма(). Условия задачи описаны в комментарии в форме. Проверить правильность решения можно встроенными тестами в режиме Предприятие.

При выполнении заданий допускается добавление/изменение существующих реквизитов объектов метаданных при условии, что не произойдет потери данных. Нужно представить, что это конфигурация рабочей базы данных, где записей в справочнике клиентов и заказов клиентов может быть > 1 млн. Таким образом, условия во всех запросах к этим таблицам должны быть только по индексированным полям.
